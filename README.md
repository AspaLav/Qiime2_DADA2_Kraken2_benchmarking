# Species-Level Benchmarking of QIIME2, DADA2, and Kraken2 Using 16S rRNA Mock Community Data

This repository contains the code and workflow documentation for a thesis project benchmarking three taxonomic classification workflows for species-level analysis of 16S rRNA mock-community data.

The workflows compared in this project are:

- QIIME2
- DADA2
- Kraken2

The aim of the project is to evaluate how closely the species-level taxonomic profiles generated by each workflow match the known ground-truth composition of two mock-community datasets.

---

## Project Overview

16S rRNA amplicon sequencing is widely used for bacterial community profiling. However, accurate classification at species level can be challenging because of sequencing errors, limited resolution of short 16S regions, reference database differences, and variation between computational workflows.

In this project, two 16S rRNA mock-community datasets with known expected species composition were analysed independently using QIIME2, standalone DADA2, and Kraken2. The predicted species lists were harmonized and compared against the known ground truth using several evaluation metrics.

The main metrics used were:

- True positives
- False positives
- False negatives
- Precision
- Recall
- F1-score
- Jaccard similarity
- Predicted species richness

A filtered Kraken2 result was also evaluated using a minimum threshold of 10 directly assigned reads.

## Workflows

### QIIME2

The QIIME2 workflow is documented in:

```text
scripts/qiime2/run_qiime2_commands.md
```

The workflow includes importing single-end FASTQ files, inspecting read quality, denoising with DADA2 single-end mode, assigning taxonomy using a Greengenes2-based classifier, and exporting taxonomy tables.

### DADA2

The standalone DADA2 workflows are stored in:

```text
scripts/dada2/run_dada2_dataset1.R
scripts/dada2/run_dada2_dataset2.R
```

The DADA2 workflow includes quality filtering, error learning, dereplication, ASV inference, chimera removal, ASV table export, FASTA export, and taxonomy assignment.

### Kraken2

The Kraken2 workflow is documented in:

```text
scripts/kraken2/run_kraken2_commands.md
```

The Kraken2 workflow includes direct read classification, extraction of species-level rows, and filtering using `Direct_reads >= 10`.

---

## Comparative Analysis

The main comparative analysis script is:

```text
scripts/analysis/compare_species_level_results.py
```

This script harmonizes species names, compares predicted species lists with the ground truth, and calculates precision, recall, F1-score, Jaccard similarity, and predicted species richness.

The main output files are:

```text
results/comparative_analysis/species_level_metrics_summary.tsv
results/comparative_analysis/species_level_metrics_details.tsv
```

## Plotting Scripts

The plots were generated using:

```text
scripts/analysis/plot_species_level_results_improved.py
scripts/analysis/plot_species_detection_heatmap.py
```

These scripts generate figures for precision, recall, F1-score, Jaccard similarity, predicted versus expected species richness, and expected species detection.

## Requirements

The project used the following tools and software versions:

- QIIME2 2024.10
- R
- DADA2
- Kraken2 2.1.3
- Python 3
- pandas
- matplotlib

Python dependencies can be installed with:

```bash
pip install pandas matplotlib
```

The standalone DADA2 scripts require R and the DADA2 package.

Large reference databases, QIIME2 classifiers, Kraken2 databases, QIIME2 artifacts, and raw FASTQ files are not included in this repository because of file size limitations. These files must be downloaded or generated separately before rerunning the full workflows.

---

## Files Not Included

The following files are excluded from the repository through `.gitignore`:

- raw FASTQ files;
- QIIME2 `.qza` and `.qzv` files;
- Kraken2 `.kraken` read-level outputs;
- Kraken2 database files;
- large reference database files;
- R intermediate `.rds` files.

## Main Finding

Across both mock-community datasets, standalone DADA2 achieved the highest F1-score and Jaccard similarity, indicating the strongest agreement with the known ground truth under the conditions tested. QIIME2 produced a more conservative classification profile, while Kraken2 generated many additional species-level predictions and required post-classification filtering.

The results should be interpreted as a comparison of complete workflows rather than a universal ranking of the tools.
